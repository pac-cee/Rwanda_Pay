You’re thinking in the right direction—but if you just ask an AI “build me an NFC payment app like Apple Pay,” you’ll get something that *looks* nice but doesn’t actually work in the real world.

Two important realities to ground this:

* NFC payments on iOS are **heavily restricted** → you can’t just access the NFC chip freely like Android
* Real tap-to-pay (like Apple Pay) requires **bank partnerships + tokenization + Apple entitlements**

So your prompt needs to be **smart and staged**:

1. Build a **realistic prototype UI + architecture**
2. Simulate payments first
3. Structure it so it *can later integrate real fintech APIs*

## 📱 Core Screens

### 1. Wallet Screen (Home)

* Shows stacked cards (like Apple Wallet)
* Swipe between cards
* Animated card expansion on tap
* Display balance and masked card number

### 2. Add Card Screen

* Form to add card (mock data)
* Camera scan simulation (optional UI only)

### 3. Payment Screen (Tap to Pay Simulation)

* Large centered card
* Pulsing animation when “ready to pay”
* NFC detection simulation:

  * When user taps button → show scanning animation
  * Then success animation (checkmark + vibration simulation)
* Show merchant name + amount

### 4. Transactions Screen

* List of transactions
* Each item:

  * Merchant name
  * Amount
  * Date
  * Status (success/pending)

### 5. Analytics Screen

* Weekly/monthly spending chart
* Categories (food, transport, etc.)
* Simple bar or line charts

### 6. Send/Receive Money

* Send via phone number or username
* Clean input UI
* Confirmation animation
